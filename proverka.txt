ОСНОВНЫЕ КОМАНДЫ git

git init 
создание репозитория в текущей папке

git add
добавление файла в репозиторий

git commit -m file_name.txt "комментарий"
фиксация текущих изменений с комментарием

git status
текущий статус git

git log
список всех коммитов

git checkout HASH
HASH - хэш, первые 6 символов предыдущих коммитов, 
вернуться в состояние предыдущих коммитов

git branch 
список всех веток

git branch new_vetka
создание новой ветки

git checkout -b new_vetka2
создание ветки и переключение на нее

git checkout main
git checkout new_vetka2
переключение на ветку main / new_vetka2

git revert HEAD
отмена последнего изменения

git merge new_vetka
слияние текущей ветки с new_vetka

git branch -d new_vetka
удаление ветки new_vetka

git clone https://github.com/magix-g/bj.git
клонирование удаленного репозитория на локальный компьютер

ПОРЯДОК РАБОТЫ С УДАЛЕННЫМ РЕПОЗИТОРИЕМ
========================================
git remote add origin1 https://github.com/magix-g/bj.git
Подключиться к удаленному репозиторию и присвоить имя соединению origin1

git branch -M main
создать ветку main

git push -u origin main
выгрузить локальный файл на удаленный репозиторий 
origin - название соединения
main - название ветки

git push
выгрузить локальный файл к уже ранее присоединенному репозиторию

git pull
скачать измененный файл с удаленного репозитория

ПОРЯДОК РАБОТЫ С PULL REQUEST
===============================================
1. Делаем форк интересующего репозитория в свой аккаунт на github.
2. Делаем клон форка на локальный комп со своего аккаунта на github.
3. Переключаемся в локальную папку форка в терминале в visual studio.
4. Переключаемся в локальную папку форка в самом visual studio.
5. Делаем новую ветку
6. В новой ветке делаем изменения. 
7. Добавляем и коммитим изменения.
8. Отправляем pull название_соединения название_ветки
9. В аккаунте на github видим эти изменения, пишем описания и отправляем кнопкой
   pull request хозяину изначального репозитория.
10. Хозяин репозитория принимает изменения. Или нет )
